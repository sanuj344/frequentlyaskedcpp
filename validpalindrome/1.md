# 📘 Valid Palindrome – Complete Interview Guide

## 🔹 Problem Statement

Given a string `s`, determine if it is a palindrome,
considering only alphanumeric characters and ignoring cases.

Example:
Input: "A man, a plan, a canal: Panama"
Output: true

---

# 🔹 Why This Question Is Important

This is a VERY common SDE Intern / SDE-1 interview question.

It tests:

* Two pointer technique
* Character handling
* Edge case management
* Clean coding under constraints

---

# 🔹 Approach 1: Brute Force

## 💡 Idea

1. Remove all non-alphanumeric characters.
2. Convert to lowercase.
3. Reverse the cleaned string.
4. Compare both strings.

## ⏱ Complexity

* Time: O(n)
* Space: O(n)

## ⚠️ Drawback

Extra space used for cleaned + reversed string.

---

# 🔹 Approach 2: Optimal (Two Pointers)

## 💡 Idea

1. Use two pointers: left and right.
2. Skip non-alphanumeric characters.
3. Compare lowercase values.
4. Move inward.

## ⏱ Complexity

* Time: O(n)
* Space: O(1)

---

# 🔹 Interview Thinking Pattern

When you see:
"Ignore cases"
"Ignore special characters"

Immediately think:

👉 Two pointers
👉 Character validation (isalnum)
👉 Case normalization (tolower)

---

# 🔹 Edge Cases to Discuss in Interview

1. Empty string → true
2. String with only special characters → true
3. Single character → true
4. Mixed uppercase & lowercase

Discussing edge cases increases interviewer confidence.

---
