# 🔹 1. What is a String?

In C++, a string is a sequence of characters stored in the `string` class from the Standard Library.

```
#include <string>
using namespace std;
```

Internally, strings behave like **dynamic character arrays**.

---

# 🔹 2. Important Concepts You Must Know

## ✅ 1. Immutability

C++ strings are mutable (unlike Java). You can modify characters directly.

```
s[0] = 'A';
```

---

## ✅ 2. Indexing

Same as arrays:

* 0-based indexing
* `s[i]`
* Out-of-bound access = undefined behavior

---

## ✅ 3. Time Complexities (Very Important for Interviews)

| Operation     | Complexity     |
| ------------- | -------------- |
| Access `s[i]` | O(1)           |
| Insert at end | O(1) amortized |
| Pop back      | O(1)           |
| Substring     | O(n)           |
| Reverse       | O(n)           |
| Compare       | O(n)           |
| Sort          | O(n log n)     |

Interviewers expect you to know these.

# 🔹 3. Patterns Used in String Problems

When solving Easy → Medium string questions, most fall into these patterns:

### 1️⃣ Frequency Counting

Use:

* Array of size 26
* Unordered map

Used in:

* Anagram
* First non-repeating character
* Valid palindrome

---

### 2️⃣ Two Pointer Technique

Used in:

* Reverse string
* Valid palindrome
* Remove duplicates

---

### 3️⃣ Sliding Window

Used in:

* Longest substring without repeating characters
* Minimum window substring

---

### 4️⃣ Sorting Based Problems

Used in:

* Anagram grouping
* Rearranging characters

---

### 5️⃣ Prefix / Pattern Matching

Used in:

* KMP (advanced)
* Rabin-Karp (advanced)

---

# 🔹 4. Interview Mindset for Strings

When you see a string problem, ask:

1. Can I use frequency array?
2. Can I use two pointers?
3. Is sliding window possible?
4. Do I need hashing?
5. Is sorting allowed within constraints?

This thinking alone solves 70% of Easy/Medium problems.
